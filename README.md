# SDF Fonts

[![Build with Alire](https://github.com/reznikmm/sdf_fonts/actions/workflows/alire.yml/badge.svg)](https://github.com/reznikmm/sdf_fonts/actions/workflows/alire.yml)
[![Alire](https://img.shields.io/endpoint?url=https://alire.ada.dev/badges/sdf_fonts.json)](https://alire.ada.dev/crates/sdf_fonts.html)

> A simple Ada library to render SDF (Signed Distance Field) fonts.

Renders bitmap text using pre-baked SDF font atlases generated by
[msdf-atlas-gen](https://github.com/Chlumsky/msdf-atlas-gen).
The library is fully generic — it renders into any discrete pixel coordinate
type via user-supplied `Draw_Pixel` and `Get_Atlas_Pixel` callbacks, with no
runtime dependencies.

## Features

- **Pure Ada** — no external runtime, no heap allocation
- **Fixed-point math only** — uses Ada fixed-point types, no floating-point operations
- **Generic renderer** — works with any pixel coordinate and distance type
- **SDF atlas support** — bilinear sampling, proper baseline alignment
- **Font packages included** — `Roboto_Mono_20` bundled and ready to use
- **Tooling included** — script to convert `msdf-atlas-gen` JSON to Ada source

## Repository Structure

- `src/` — Library sources
  - `sdf_fonts.ads` — Core types (`Glyph`, `Plane_Bounds`, `Atlas_Bounds`, …)
  - `sdf_fonts-render_glyph.ads/adb` — Generic glyph renderer
  - `sdf_fonts-roboto_mono_20.ads/adb` — Bundled Roboto Mono font at size 20
- `examples/` — Examples applications (ASCII art renderer)
- `scripts/json_to_ada.sh` — Converts `msdf-atlas-gen` JSON output to Ada
- `alire.toml` — Alire crate manifest

## Installation

Add the crate via [Alire](https://alire.ada.dev):

```shell
alr with sdf_fonts
```

## Usage

### 1. Instantiate the renderer for your font package

```ada
with SDF_Fonts.Roboto_Mono_20;

procedure My_App is
   Screen : array (1 .. 30) of String (1 .. 80) :=
     (others => (others => ' '));

   procedure Draw_Pixel (X, Y : Positive) is
   begin
      Screen (Y) (X) := '#';
   end Draw_Pixel;

   procedure Render_Glyph is new SDF_Fonts.Roboto_Mono_20.Render_Glyph
     (Positive, Draw_Pixel);

   X : Positive := 5;
begin
   --  Render glyphs by sequential index (0-based, ordered as in the atlas)
   Render_Glyph (X => X, Y => 15, Scale => 22, Index => 32);  --  'A'
   Render_Glyph (X => X, Y => 15, Scale => 22, Index => 67);  --  'd'
   Render_Glyph (X => X, Y => 15, Scale => 22, Index => 64);  --  'a'
end My_App;
```

`X` is updated after each glyph by the advance width, so you can render a
string left-to-right by calling `Render_Glyph` in sequence.  `Y` is the
**baseline** row in pixel coordinates.

## Creating a New Font

Follow these steps to bake a new font into an Ada package.

### Step 1 — Generate the atlas with `msdf-atlas-gen`

Install [msdf-atlas-gen](https://github.com/Chlumsky/msdf-atlas-gen).
Prepare a charset file (e.g. `ascii.txt`) listing the desired Unicode ranges:

```
['!', '~']
```

Run the generator:

```sh
bin/msdf-atlas-gen \
  -font /usr/share/fonts/truetype/myfont/MyFont.ttf \
  -charset ascii.txt \
  -type sdf \
  -format png \
  -imageout myfont.png \
  -json myfont.json \
  -csv myfont.csv \
  -size 20 \
  -dimensions 128 128
```

Adjust `-size` and `-dimensions` to taste. Larger sizes give sharper glyphs
at the cost of atlas memory.

### Step 2 — Convert the atlas image to raw bytes

```sh
ffmpeg -i myfont.png -f rawvideo -pix_fmt gray myfont.raw
```

### Step 3 — Format the raw bytes as an Ada array literal

```sh
hexdump --format '"(" 128/1 "+%u," "),\n"' myfont.raw
```

Adjust the row width (`128`) to match your `-dimensions` width.

### Step 4 — Convert the JSON glyph metadata to Ada case arms

```sh
scripts/json_to_ada.sh myfont.json
```

Output looks like:

```ada
         when 1 =>
        (0.60205078125,
         (0.176513671875, 0.426513671875, 0.84375, -0.09375),
         (42.5, 46.5, 95.5, 80.5)),
```

Each `when` arm corresponds to one glyph in atlas order (0-based).
Fields are: `(Advance, (Left, Right, Top, Bottom), (Left, Right, Top, Bottom))`.

### Step 5 — Write the Ada font package

Create `src/sdf_fonts-my_font.ads`:

```ada
package SDF_Fonts.My_Font is

   function Get_Glyph (Index : Glyph_Index) return Glyph;

   type Distance is delta 1.0 / 128.0 range -128.0 .. 127.0;

   function Get_Atlas_Pixel (X, Y : Atlas_Coordinate) return Distance
     with Inline;

   generic
      type Pixel_Coordinate is (<>);
      with procedure Draw_Pixel (X, Y : Pixel_Coordinate);
   procedure Render_Glyph
     (X     : in out Pixel_Coordinate;
      Y     : Pixel_Coordinate;
      Scale : Font_Scale;
      Index : Glyph_Index);

end SDF_Fonts.My_Font;
```

Create `src/sdf_fonts-my_font.adb` and fill in:

- `Get_Glyph` — a `case` expression using the output from Step 4
- `Get_Atlas_Pixel` — indexes into the raw atlas bytes from Step 3
- `Render_Glyph` — instantiation of `SDF_Fonts.Render_Glyph`

See `src/sdf_fonts-roboto_mono_20.adb` for a complete working example.
